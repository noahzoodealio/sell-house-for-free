Named exports (<code>export const HOME = '/'</code>) pollute the file's public surface. <code>LINKS.city(slug)</code> is a function inside the same object, keeping the API unified.</li><li><strong>Why <code>as const</code>.</strong> Without it, each entry is typed as <code>string</code>. With it, each entry is typed as its literal (<code>'/listing'</code> is of type <code>'/listing'</code>), enabling typed unions in downstream consumers (e.g., <code>type PillarPath = typeof LINKS.listing | typeof LINKS.cashOffers | …</code>). Not immediately needed but cheap.</li><li><strong>Why the home's FAQ excerpt filters to <code>free-and-fair</code> or <code>skepticFirst</code>.</strong> The home has limited visual real estate. "How is this free?" + "What's the catch?" + "Will you sell my info?" are the highest-leverage objections. Showing more dilutes attention; showing fewer leaves objections on the floor.</li><li><strong>Why the home emits FAQPage JSON-LD for the excerpt.</strong> Schema.org FAQPage eligibility requires the schema to match visible content. Excerpt → excerpt schema. Full <code>/faq</code> page emits the full <code>entries</code> array via <code>faqPageSchema(entries)</code>; home emits the filtered subset. Both pages can be rich-results-eligible if validators approve.</li><li><strong>Why pillar blurbs colocate in the home file, not a shared constant.</strong> Each pillar's landing page (S8) will ship its own longer copy in its own file. The home's 1-sentence blurb is a teaser distinct from the landing page's full treatment. Duplication at this granularity is fine; sharing would tie two surfaces' wording together unnecessarily.</li><li><strong>Why the sitemap picks up <code>/get-started</code> via E1-S5, not this story.</strong> E1-S5 already registered <code>/get-started</code> in <code>routes.ts</code>. Don't add it here or you'll duplicate-entry in the sitemap. Double-check <code>routes.ts</code> before appending.</li><li><strong>City registry priority.</strong> All seven AZ city slugs get <code>priority: 0.7</code> — same tier as <code>/faq</code> and <code>/meet-your-pm</code>. They're SEO landers, not the canonical highest-priority surfaces. If analytics post-launch show city landers driving conversions, raise to 0.8 in a follow-up.</li><li><strong>Why the home's <code>title.absolute</code>.</strong> Without it, Next.js 16 appends the E1-S1 template suffix ("| Sell Your House Free"), producing "Sell Your House Free — Free Arizona cash-offer service, no fees, real PM | Sell Your House Free" — duplicated brand. The <code>absolute</code> variant suppresses the template for this page only. The CTA IS <code>/get-started</code>; adding a contact form splits intent.</li><li><strong>Route registry is ambiguously owned.</strong> <code>routes.ts</code> was created by E1-S2; edits from E2 are appends.
