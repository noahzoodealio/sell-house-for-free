# E6-S4 — Resend + React Email

Replaces the S3 email stub with real Resend wiring. pm-service barrel now points at `@/lib/email`; `email-stub.ts` deleted.

## Key decision: timeout via Promise.race, not AbortSignal

Resend v6 SDK's `emails.send()` does not accept an AbortSignal (not in `CreateEmailRequestOptions`). Implemented the 3s per-attempt timeout by racing against a `TimeoutError` promise. The caller resolves at 3s even if the upstream fetch keeps running — each retry writes its own `notification_log` row so a late-completed attempt is detectable by provider_message_id reconciliation. This is the compromise called out in the scope revision as Vercel runtime-termination risk; acceptable for MVP.

## Sentry event contract unchanged

`pm_email_failed` is still emitted from the pm-service orchestrator when an email returns ok:false. The send module itself doesn't emit Sentry — it returns a `SendResult`. Orchestrator decides to alert.

## PII discipline

`sanitizeError` allow-list in `send.ts:SENSITIVE_KEYS`:
- Drops: `to`, `from`, `replyTo`/`reply_to`, `email`, `phone`, `address`, `headers`
- Keeps: `statusCode`, `status`, `code`, `message`, and other string/number/boolean fields
- Truncates final output to 500 chars

Team-member template is the only email surface that carries seller PII. Seller template exposes only the seller's first name, the PM's first name + photo, and the referral code.
