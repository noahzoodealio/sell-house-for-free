## Decisions

- E2E regexes relaxed from `\\?ref=${submissionId}$` to `\\?sid=${submissionId}(?:&|$)`. Explanation: the old regex matched `ref=${submissionId}` — a vestige from an earlier redirect URL design. Current E5 redirect is `/portal/setup?sid=<uuid>&ref=<offervanaReferralCode>`; asserting the Offervana referral code's exact value couples the test to a mock detail. Asserting `sid=<submissionId>` is enough to prove the round trip.
- Kept the existing `<PortalLoading />` polling island untouched. S6 only adds a server-rendered block above it.
- `ref` type-narrow with `typeof !== 'string'` fallthrough — arrays, undefined, and missing all collapse into the fallback render.
- CopyButton is a thin client island (12 lines) rather than converting the whole page. SubmissionRef stays server.

## Potentially fragile

The E2E specs exercise the actual client-side flow including `PortalLoading` polling against `/api/portal/submission-status`. Haven't re-run E2E end-to-end with real Supabase + Offervana — the tests may need adjustments when run in CI against the live stack. Deferred to a human CI pass.
