### Next.js 16 conventions relevant to E1

- **`params` / `searchParams` are `Promise<…>`** — must be awaited. (Breaking vs. Next 14.)
- **`PageProps<'/path'>` / `LayoutProps<'/path'>`** are globally available type helpers generated by `next typegen`. Use them; don't hand-type routes.
- **Route groups** `(group-name)` — parenthesized folders don't affect URL. Use to give marketing / legal / funnel sections their own layouts.
- **File-based metadata:** `app/favicon.ico`, `app/icon.{ts,tsx}`, `app/opengraph-image.{ts,tsx}`, `app/robots.{ts,txt}`, `app/sitemap.{ts,xml}`, `app/manifest.{ts,json}`. Generated files are Route Handlers cached at build time unless they touch request-time APIs.
- **Streaming metadata** is default for dynamically rendered pages — fine for MVP but the SSR-prerendered marketing pages will resolve at build time (no stream).
- **`metadataBase`** must be set on the root `layout.tsx`'s `metadata` export so relative OG image URLs resolve correctly.
- **`unstable_instant`** — validated-instant route segment export, tied to Cache Components. Mentioned in root docs hint but **not adopted in E1**. Reason: it requires Cache Components mode, which changes data-fetching semantics across the app. Defer to a dedicated decision later (likely E4 or post-launch). E1 sticks with default Suspense + streaming.
- **Tailwind v4** — theme config lives inside `globals.css` via `@theme` blocks (which the scaffold already uses). No `tailwind.config.ts` file.
- **`next/font`** — self-hosted, no browser-to-Google request. Aligns with our no-PII-to-third-parties posture.

### Design tokens — mapped into Tailwind v4 `@theme`

`src/app/globals.css`:

```css
@import "tailwindcss";

@theme {
  /* Brand */
  --color-brand: #0653ab;
  --color-brand-foreground: #fdfdfd;

  /* Ink (headings / body / muted) */
  --color-ink-title: #17233d;          /* Headings — deep navy */
  --color-ink-body: #212121;           /* Body copy */
  --color-ink-muted: #9e9e9e;          /* Placeholder / muted */
  --color-ink-disabled: #bdbdbd;

  /* Surfaces */
  --color-surface: #ffffff;
  --color-surface-tint: #fafafa;
  --color-surface-dark: #1f1f1f;       /* Button label on secondary */

  /* Borders */
  --color-border: #bdbdbd;
  --color-border-strong: #0653ab;

  /* Radius */
  --radius-sm: 4px;
  --radius-md: 6px;
  --radius-lg: 8px;                    /* Buttons + cards default */
  --radius-xl: 12px;

  /* Shadow (Figma: Light/Shadow-6) */
  --shadow-elevated: 0 8px 16px 0 #60617029, 0 2px 4px 0 #28293d0a;

  /* Fonts */
  --font-sans: var(--font-open-sans), system-ui, sans-serif;   /* body */
  --font-display: var(--font-inter), system-ui, sans-serif;    /* headings */

  /* Container widths */
  --container-prose: 65ch;
  --container-form: 560px;
  --container-page: 1280px;
}
```

**Why semantic names, not scale names.** We name the role (`--color-ink-title`) rather than the value (`--color-navy-700`) so that a later palette refresh touches this file only, not every component.

### Typography strategy

Two `next/font/google` calls in `app/layout.tsx` exposed as CSS variables:

```tsx
import { Inter, Open_Sans } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
  display: "swap",
});

const openSans = Open_Sans({
  subsets: ["latin"],
  variable: "--font-open-sans",
  display: "swap",
});

// <html className={`${inter.variable} ${openSans.variable}`}>
```

`--font-display` (Inter) is applied to `h1-h6` via the base layer. `--font-sans` (Open Sans) is the body default.

**Size ramp:** instead of memorized tokens, use Tailwind utilities anchored to the Figma spec — e.g. `text-[44px] md:text-[80px]` on H1, `text-[18px] md:text-[20px] font-semibold` on primary body. Type sizes are not exposed as tokens because the ramp is breakpoint-dependent (H1 is 44 mobile / 80 desktop) and mapping to single-value tokens obscures that.

### Component primitives — structure

All primitives are **small typed wrappers around native elements**. No Radix. No shadcn/ui. Constraint: every component must pass through native `...props` so forms (E3) can wire up `name`, `value`, `onChange`, `aria-*` without us re-implementing them.

**Variants:** local utility per component, not `class-variance-authority`. Example:

```tsx
// src/components/ui/button.tsx
type Variant = "primary" | "secondary";
type Size = "xs" | "sm" | "md" | "lg" | "xl";

const VARIANT: Record<Variant, string> = {
  primary: "bg-brand text-brand-foreground hover:bg-brand/90",
  secondary: "bg-surface text-surface-dark border-[1.5px] border-brand hover:bg-brand/5",
};

const SIZE: Record<Size, string> = {
  xs: "h-[42px] px-4 text-[16px]",
  sm: "h-[48px] md:h-[52px] px-5 text-[18px]",
  md: "h-[48px] md:h-[52px] px-6 text-[18px]",
  lg: "h-[48px] md:h-[52px] px-8 text-[18px]",
  xl: "h-[48px] md:h-[52px] px-10 text-[18px]",
};
```

(Illustrative — exact widths in the Figma spec are fixed pixel values tied to specific mocks; in code we size by padding + content so buttons flex to content.)

**Focus ring:** `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-brand` on every interactive primitive. Non-negotiable for WCAG.

**Field pattern:**

```tsx
<Field label="Email" htmlFor="email" error={errors.email}>
  <Input id="email" type="email" name="email" required />
</Field>
```

`Field` owns: label-input association, error text rendering, `aria-invalid`, `aria-describedby`. Consumed by E3's form steps.

### SEO defaults

**`src/lib/site.ts`** — single source of truth:
```ts
export const SITE = {
  name: "Sell Your House Free",
  url: process.env.NEXT_PUBLIC_SITE_URL ?? "http://localhost:3000",
  description: "List your Arizona home on MLS and all major platforms …",
  locale: "en_US",
  region: "AZ",
  phone: "…",        // placeholder until E7
  broker: "JK Realty", // per plan Q2
};
```

**`src/app/layout.tsx`** — root metadata:
```ts
export const metadata: Metadata = {
  metadataBase: new URL(SITE.url),
  title: { default: SITE.name, template: `%s | ${SITE.name}` },
  description: SITE.description,
  openGraph: { type: "website", siteName: SITE.name, locale: SITE.locale },
  twitter: { card: "summary_large_image" },
  robots: { index: SITE_ENV === "production", follow: SITE_ENV === "production" },
};
```

**`src/app/robots.ts`** — env-aware: allow `/` in production, `disallow: /` on preview so Vercel preview URLs don't get indexed.

**`src/app/sitemap.ts`** — reads a route registry from `src/lib/routes.ts` (array of `{ path, changeFrequency, priority }`). E2 adds marketing pages to the registry, E7 adds legal pages, E3 adds `/get-started`. The sitemap function does no network I/O — static build-time output.

**`src/app/opengraph-image.tsx`** — `ImageResponse` that renders the brand name + tagline on the brand blue. Provides a fallback OG image for every route; pages can override by placing their own `opengraph-image.tsx`.

### Analytics gating

Scaffold already has `<Analytics />` from `@vercel/analytics/next`. Wrap:

```tsx
{process.env.NODE_ENV === "production" && <Analytics />}
```

And **do not add** any other analytics library in E1. Document in `AGENTS.md` (or a new `docs/analytics-policy.md`) that third-party analytics SDKs are prohibited without sign-off — enforces the "we don't sell your data" promise structurally, not just in copy.

### Dark mode — dropped

The scaffold `globals.css` has `@media (prefers-color-scheme: dark) { … }`. Remove. Rationale:
- Plan does not require dark mode.
- Brand tokens are designed for light surfaces.
- Cuts design-review + visual-regression surface in half for E2/E7.

Tradeoff accepted: some mobile users with OS dark mode will see our light site. Acceptable for an AZ real-estate audience.

### CSP / security headers — deferred to E8

The `data-security.md` guide in Next.js 16 is worth a read at E8 launch, but adding strict CSP in E1 tends to slow iteration (third-party Vercel Analytics / font-preload all need nonce work). E1 scope: **no CSP**. E8 scope: add CSP + other headers in `next.config.ts` `headers()`.

### Pattern decisions & deviations

| Decision | Pattern cited | Deviation? |
|---|---|---|
| App Router + `src/app/` | Next.js 16 `src-folder.md` + scaffold | No |
| Tailwind v4 `@theme` in CSS | Next.js 16 `css.md` §Tailwind + scaffold | No |
| `next/font` self-hosted Inter + Open Sans | Next.js 16 `fonts.md` | No |
| Route groups `(marketing)`, `(legal)` | Next.js 16 `route-groups.md` | No |
| File-based metadata (`sitemap.ts`, `robots.ts`, `manifest.ts`, `opengraph-image.tsx`) | Next.js 16 metadata file conventions | No |
| Handrolled primitives (no Radix) | — (no prior art) | Justified: minimal E3 form needs, can add Radix later per-control if needed |

| Drop `unstable_instant` | Next.js 16 hint in index.md | Justified: requires Cache Components, too invasive for E1 |
| Semantic token names (`--color-ink-title`) | — | Project-specific; preserves rename headroom |

## Step 4 — Integration contracts (in progress)

E1 has **no cross-service calls**. "Integration contracts" here means the extension points later epics plug into.

| Consumer epic | Hook | Contract |
|---|---|---|
| E2 (marketing pages) | `src/app/(marketing)/` + `src/lib/routes.ts` | Add `page.tsx` under `(marketing)/...`. Register canonical URL in `routes.ts` so sitemap + nav pick it up. Use `buildMetadata()` from `src/lib/seo.ts`. |
| E3 (submission form) | `src/app/get-started/page.tsx` + `FormStep`, `Field`, `Input`, `Select`, `Checkbox` primitives | Replace placeholder page. Form state is owned by E3; E1 provides no form state library. |
| E4 (enrichment BFF) | `src/app/api/` (not created in E1) + `src/lib/` for MLS client | E1 does not scaffold `api/`. E4 creates route handlers at `app/api/enrich/route.ts`. |
| E5 (Offervana submission) | Same `src/app/api/` | Same. |
| E6 (PM confirmation) | `src/app/confirmation/[referralCode]/page.tsx` (E6 creates) + `lib/supabase.ts` (E6 creates) | E1 does not install `@supabase/supabase-js`. |
| E7 (compliance) | `src/app/(legal)/` | Legal routes use the `(legal)` group's layout. |
| E8 (launch) | `next.config.ts` `headers()` + `src/app/robots.ts` (prod gating already wired) | E1 wires env-aware robots; E8 adds CSP / HSTS / security headers. |

Env vars introduced in E1:
- `NEXT_PUBLIC_SITE_URL` — base URL (used by `metadataBase`, sitemap, robots)
- `SITE_ENV` (or rely on `VERCEL_ENV`) — `production` | `preview` | `development`; drives robots indexability
